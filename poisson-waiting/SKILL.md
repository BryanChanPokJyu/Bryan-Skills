# 泊松等待纪律（Poisson Waiting Discipline）

投资不是连续收益函数，而是**稀疏事件驱动的状态跳变过程**。大部分时间什么都不发生，真正改变结果的是少数关键事件突然到来。

优秀投资人不是每天行动，而是识别 λ 正在上升的地方，在跳变前完成准备，并在长时间无反馈中不被情绪和噪音摧毁。

> 买的不是股票，买的是 λ；等的不是时间，等的是证据网络达到临界密度的那一刻。

这是决策纪律框架，不是价格预测器，也不替代用户自行作出的投资决策。

## 核心心智模型

- **泊松过程**：事件稀疏、随机、间歇到来。`N(t) ~ Poisson(λt)`。
- **λ = 向上状态跳变的强度**：表示未来一段时间内发生向上状态跳变的概率密度，不表示明天涨不涨。
- **等待时间服从指数分布**，均值为 `1/λ`：λ 越高，平均等待越短。
- **无记忆性**：已经等很久，不代表下一个机会更快到来。「跌够久该涨了」是等待时间幻觉，不是证据。

真实市场更接近 Hawkes 过程：事件会自我激发、成簇出现，λ 会随时间变化。每次复盘都必须重估 λ 方向，不要假设 λ 恒定。

## 铁律

每次判断都必须遵守：

1. 不要输出假精确的 λ 数值。λ 不可直接观测，只使用有序档位：**低 / 上升 / 高 / 下降**。每个档位必须附证据。不要写 `λ=0.73`。
2. 说不清在等什么具体事件，就不是等待，而是套牢后的自我安慰。
3. 单条新闻或管理层口号不是证据网络。必须由多个独立信号交叉印证。
4. 区分价格噪音、证据恶化、状态迁移失败和机会成本上升。价格噪音可以等；证据恶化不能用长期主义麻醉。
5. 警惕无事件期过度行动：不要用小新闻替代大证据，不要因无聊换股，不要因为等急了就把噪音当催化剂。

## 工作流

### 第 0 步：拉取实时数据

不要使用记忆里的旧数据。价格、估值、最新财报增速等字段必须获取实时数据，否则卡片会过期。

```bash
python3 ~/.claude/skills/poisson-waiting/scripts/fetch-data.py <TICKER> --news 5
```

脚本使用 yfinance 拉取客观快照，包括价格、PE/PS、目标价、毛利率、收入增速、下次财报日和近期新闻，并输出半填好的卡片 JSON：

- 客观字段放在 `_data_snapshot`。
- 主观判断字段为 `FILL:` 或 `null`。
- 数据源为 Yahoo，仅用于个人研究。
- 价格实时性通常为分钟级到天级，基本面为最近一期财报。

若脚本失灵，优先检查数据源字段变化并修复脚本。

### 第 1 步：检查历史卡片

这个框架的价值在持续复盘，不在单次诊断。

先在飞书里查询该标的是否已经存在历史卡片。使用 lark-doc / lark-wiki skill 搜索标的名和 `Poisson`。

- **有历史卡片**：重点比较「自上次复盘以来，证据网络变密了，还是只有价格在动？」
- **无历史卡片**：创建新卡片。

### 第 2 步：生成卡片

读取第 0 步输出的 JSON，并结合 `_data_snapshot` 里的实时数据填完所有 `FILL:` 和 `null` 主观字段：

- 填写日期。
- 填写正在等待的具体事件。
- 判断 λ 方向。
- 判断六项证据真假。
- 填写赔率、证伪条件和叙事自检。
- 删除或保留 `_data_snapshot` 均可。

用数据做锚点：

- 用 revenue YoY 是否仍在加速判断 `revenue_accelerating`。
- 用毛利率是否恶化判断 `margin_intact`。
- 用 forward PE 和 `implied_upside_pct` 判断 `price_not_reflected`、`upside_odds` 和 `downside_loss`。

确定性计算必须交给脚本，不要手算证据密度、四象限、EV/单位时间分档或叙事麻醉警报。

```bash
# 查看输入模板
node ~/.claude/skills/poisson-waiting/scripts/build-card.js --schema

# 填好 JSON 后生成卡片，可使用 stdin 或 --in 文件
echo '<填好的 JSON>' | node ~/.claude/skills/poisson-waiting/scripts/build-card.js
```

脚本应自动完成：

- 按六项证据数判断「远离 / 接近 / 已达临界密度」。
- 按 λ × 赔率判断四象限。
- 按 λ、证据和成本综合判断 EV/单位时间正负。
- 检测死扛成本价、用价格当证据、无 falsifier、「跌够久了」式幻觉等叙事麻醉信号。
- 强制 λ 使用有序档位，禁止数值化。

### 第 3 步：补充结论

脚本生成卡片后，补充：

- 当前动作：`准备建仓 / 加仓 / 持有等待 / 减仓 / 退出 / 仅观察`
- 一句话理由：必须落在 λ 和证据上，不能落在价格或情绪上。
- Review Clock：写明下次复盘时间或触发条件。

不要手算 EV 数字。以下逻辑已由脚本按档位定性判断：

```text
EV / 单位时间 =
λ × P(真迁移 | 事件) × Payoff
− Carry
− Fragility
− Opportunity
− FalsePositive
```

### 第 4 步：写回飞书并提示复盘

使用 lark-doc skill 将卡片写入飞书文档，以便团队共享和持续复盘。

明确告诉用户 Review Clock。到点或触发条件发生时，主动提醒用户重新评估。

## 卡片结构

```markdown
# Poisson Waiting Card — {标的} — {日期}

## 1. 在等什么事件
具体写：下季财报 beat？大客户订单确认？毛利率反转？行业价格反转？指数纳入？
写不出具体事件 → 标记「⚠️ 无明确事件，不是等待」并停止，不要硬凑。

## 2. λ 方向
[低 / 上升 / 高 / 下降] + 支撑证据

- 低：催化稀疏，长期等待，小仓位或仅观察
- 上升：状态迁移概率提高，值得加大研究和准备
- 高：催化密集，可能进入重定价窗口
- 下降：thesis 进入低反馈期，重新评估机会成本

## 3. 证据网络（是否接近临界密度）
- [ ] 客户在买 / 采用率上升
- [ ] 收入在加速
- [ ] 利润率没崩
- [ ] 订单 / RPO / backlog 增长
- [ ] 行业趋势配合
- [ ] 价格还没完全反映

→ 判定：远离 / 接近 / 已达临界密度

## 4. 赔率不对称
上行赔率 ÷ 下行损失 = ?
非线性右尾是关键。

## 5. 证伪条件（falsifier）
- 什么情况说明我错了：
- 什么情况说明没有进入状态跳变：
- 什么情况说明等待已变成资本钝化：

## 6. 四象限定位
λ（低 / 高）× Payoff（低 / 高）
警惕：不要把「低 λ + 高 payoff」错配成核心仓位。

## 7. 叙事麻醉警报
检查并标红：无证据更新 / 无 falsifier / 因成本价死扛 / 用「跌够久了」当理由 / 用价格波动替代证据更新。

## 8. 结论 + Review Clock
- 当前动作：[准备建仓 / 加仓 / 持有等待 / 减仓 / 退出 / 仅观察]
- 一句话理由：
- 下次复盘时间 / 触发条件：
```

## 数据新鲜度

### 🔴 必须实时

过期会翻转结论，由 `fetch-data.py` 拉取：

- 当前价格和估值
- 最新一季收入增速
- 毛利率
- 赔率所需的估值水平与隐含上行
- 最近的大客户 capex 和财报新闻

### 🟡 半实时

按月或按季更新：

- 下次财报日
- 新品代际节奏
- 客户自研芯片替代进度

### 🟢 无需实时

逻辑结构可以复用：

- 证伪条件
- 四象限
- 叙事麻醉警报
- EV 分档
- Review Clock

## 与新闻因子引擎的预留接口

λ 的本质是事件到达强度，与公司的 EVI / ETR 新闻因子引擎是同一件事的两种表述。

当前阶段由人工和 Claude 评估 λ。卡片第 2 步预留 `EVI/ETR` 输入字段，未来可以直接接入引擎输出；现在不要打通。

## 重要提醒

- 这是决策纪律框架，不是预测器。产出是逼用户回答难问题的结构化判断，不是「建议买入」。
- 分析判断由 Claude 完成；仅在涉及英文材料翻译时使用 GLM。
- 每次复盘都要重估 λ 方向。
